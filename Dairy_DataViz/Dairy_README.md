# Team Dairy: AAEA Data Visualization Challenge

This repository documents and reproduces two visualizations created for our AAEA Data Visualization Challenge submission:

- **Visualization 1: Local-Market Channels in Dairy-Producing Counties**

- **Visualization 2: Milk Production, Processing, and Diversity**

The repository is organized into two main proejct folders:

- `aaea_dairy_viz_1/`
- `aaea_dairy_viz_2/`

Each folder contains the data, code, and outputs needed to reporduce its corresponding visualization. The sections below describe the data sources, processing decisions, code, outputs, and replication steps for each visualization.

## Repository Structure
### aaea_dairy_viz1

Visualization 1 is organized into the following subfolders:

- `raw/` folder contains the original FAME, Census of Agriculture, food-hub, and county-boundary files used to construct the Dairy Market Pathways Index.

- `outputs/` contains the final maps and supporting county-level data files generated by the code.

Visualization 1 does not have a separate `processed/` folder. Its processing steps are performed directly within the Python script `code/Dairy_DataVisualization1.py`. The resulting county-level analytical dataset `Dairy_DataVisualization1.xlsx` and visualization `Dairy_DataVisualization1.png` are saved in `outputs/`.

### aaea_dairy_viz2

Visualization 2 is organized into the following subfolders:

- `raw/` contains the original and downloaded source files used by the cleaning notebook. These include the FMMO producer-milk reports, PDF-to-text table extracts, annual FMMO class-utilization report, December 2025 USDA AMS plant-table downloads, Census county boundaries, city-to-county crosswalk, county coordinates, and federal marketing-area definitions. Files in this folder are not modified during the analysis.

- `processed/` contains intermediate files created from the raw sources by `code/Dairy_data_cleaning.ipynb`. These include the parsed producer-milk tables for each FMMO, the combined county-level milk dataset, reconciliation checks against published totals, geocoded supply plant list and geocoded distributing plant list, annual class-utilization shares, and county lists used to draw the FMMO marketing-area boundaries. These files are included to make the transformations transparent, but they can be recreated by running the cleaning notebook.

- `code/` contains the two Jupyter notebooks used to reproduce the analysis. `Dairy_data_cleaning.ipynb` downloads or reads the cached plant data, parses and combines the FMMO reports, geocodes the plant records, creates the marketing-area files, and saves the intermediate datasets in `processed/`. `Dairy_DataVisualization2.ipynb` reads the processed datasets, joins them to the geographic files, and produces visualization 2.

- `outputs/` contains the final file generated by the visualization notebook: `Dairy_DataVisualization2.png`.

The following sections provide detailed information about the data sources and processing steps used for each visualization.

## Visualization 1: Dairy Local Market Channel Ecosystem

### Data Sources

- `raw/fame_master_feb2026.csv`\
  FAME 2.0 county-level master data, February 2026 file. The file contains many food and agricultural market variables. The script filters this file to 2022 county-level `d2c_sales_pct` and `intermediated_sales_pct`. The most recent available data is for 2022.

- `raw/foodhub_2026.xlsx`\
  Food hub records downloaded from the USDA Local Food Portal: https://www.usdalocalfoodportal.com/. This is a 2026 portal snapshot rather than an annual Census-style dataset; individual listing update timestamps in the file range from 2014 to 2026. The script keeps food hubs that report local or partly local dairy sourcing and valid coordinates.

  The food hub measure should be interpreted as portal-listed local dairy food hub presence, not as a verified count of currently operating food hubs. The file includes listing update timestamps, but it does not provide a confirmed active/closed operating status.

- `raw/dairy_cows_quickstats_2022.csv`\
  USDA NASS Quick Stats records from the 2022 Census of Agriculture for county-level `CATTLE, COWS, MILK - INVENTORY`. The script uses reported inventory values when available and also parses the lower bound of reported inventory classes so counties with suppressed totals can still be screened when the class indicates at least 100 milk cows.

- `raw/county_boundaries.zip`\
  U.S. Census Bureau 2023 county cartographic boundary shapefile. The script uses this file to draw county shapes and calculate county representative points. Counties are projected to EPSG:5070 before distance calculations.

### Data Processing

`code/Dairy_DataVisualization1.py` starts from the raw files. It filters FAME to the two sales-channel variables used in the index, summarizes the Quick Stats dairy-cow records into a county-level dairy-base screen, filters food hubs to local or partly local dairy sourcing records, and joins all county-level inputs to Census county boundaries by county FIPS code.

For each county, the script counts retained local dairy food hubs within 75 miles of the county representative point. The index is built among counties passing the 100+ milk-cow screen. The three components are standardized as z-scores within that dairy-base universe: direct-to-consumer sales share, intermediated local-channel sales share, and nearby local dairy food hub count. The final index is the simple average of the three standardized components only when all three are present.

No missing index components are imputed. Counties outside the dairy-base screen or missing any index component are shown in grey. The underlying index values are not censored; only the map color scale is capped at the 95th percentile so high outliers do not dominate the visual scale.

### Outputs

- `code/Dairy_DataVisualization1.py`\
  Reproduces Visualization 1 from the raw files.

- `outputs/Dairy_DataVisualization1.png`\
  Final two-panel map comparing the Upper Midwest (MI, MN, WI) and the Northeast (NY, PA, VT).

- `outputs/Dairy_DataVisualization1.xlsx`\
  County-level table used to draw the map, including input variables, the dairy-base flag, food hub count, standardized components, final index, and color-cap indicator.

### Replication Instructions

From the `aaea_dairy_viz_1` folder, run: `Dairy_DataVisualization1.py`.

The script uses paths relative to its own location, reads from `raw/`, and writes to `outputs/`, so users do not need to edit local folder paths after downloading the repository. The Python environment needs standard geospatial and data-analysis packages: `pandas`, `geopandas`, `matplotlib`, `numpy`, and `openpyxl`.

## Visualization 2: Milk Production and Processing

### Data Sources and Data Processing

#### Data Source 1: Federal Milk Marketing Order (FMMO) Producer-Milk Reports

- `raw/FMMO1_northeast_dec2025.csv`

- `raw/FMMO5_appalachian_dec2025.pdf`

- `raw/FMMO30_uppermidwest_dec2025.pdf`

- `raw/FMMO32_central_dec2025.pdf`

- `raw/FMMO33_mideast_dec2025.pdf`

These files were downloaded from the statistical-report pages maintained by the five FMMO official websites. All files contain data for December 2025. These files are available in `raw/` and the URLs to access the data directly from the FMMO websites can be found below:

- **FMMO 1 — Northeast Marketing Area:**\
  [Statistical Reports](https://www.fmmone.com/Statistical_Report.htm)

- **FMMO 5 — Appalachian Marketing Area:**\
  [2025 Annual Statistical Report](https://www.malouisville.com/statistical_material/annual_report?2025)

- **FMMO 30 — Upper Midwest Marketing Area:**\
  [Producer Milk by State and County](https://www.fmma30.com/ProducerData.html)

- **FMMO 32 — Central Marketing Area:**\
  [Statistical Reports](https://www.fmmacentral.com/statistical_reports.html)

- **FMMO 33 — Mideast Marketing Area:**\
  [Statistical Reports](https://www.fmmaclev.com/Statistical_Report.htm)

**Raw data:** The reports provide producer milk pooled by state and county, generally reported in pounds, along with the number of participating producers where disclosure rules allow. The five orders were included because milk originating in the six study states may be pooled on different FMMOs. For example, most pooled milk reported for Michigan is associated with FMMO 33 rather than FMMO 30.

For FMMO 1, we save the "Producer Receipts by State and County" table from the order's 2025 Annual Statistical Bulletin as a text-based CSV, `raw/FMMO1_northeast_dec2025.csv`. The other reports were text-based PDFs. We converted them to text files using Poppler’s pdftotext -layout tool, which helps preserve the spacing and columns in the original tables.  The converted text files can be found in `\raw` with the following file names:

- `raw/FMMO5_dec2025_extract.txt`

- `raw/FMMO30_dec2025_extract.txt`

- `raw/FMMO32_dec2025_extract.txt`

- `raw/FMMO33_MI_dec2025_extract.txt`

- `raw/FMMO33_NY_PA_dec2025_extract.txt`

`code/Dairy_data_cleaning.ipynb` reads these table extracts and organizes each row into state, county, FIPS code, producer count, and pounds of milk. We keep records for Michigan, Minnesota, Wisconsin, New York, Pennsylvania, and Vermont, which are the six states of interest for the two regions in our visualizations.

When a county had milk reported by more than one included order, its order-specific quantities were summed rather than assigning the county to only one order. Published order-state totals were retained to check the extraction and aggregation. Suppressed county values were not imputed and are displayed in gray in the visualization.

**Importance for the visualization**: These reports provide the only consistent county-level measure available for this analysis of where milk participating in the FMMO system originated. USDA reports that FMMOs cover approximately 75% of U.S. milk production, making the reports a broad, though incomplete, source of information on regulated milk markets. The reports do not show where the milk was processed or sold. Therefore, we use the data only to establish the geographic origin of reported producer milk.

#### Data Source 2: USDA AMS Regulated Plant Lists

We use the following national lists of regulated pool supply plants and distributing plants:

- [Regulated Pool Distributing Plants by Month](https://mpr.datamart.ams.usda.gov/services/v1.1/reports/3359/Distributing%20Plants%20by%20Month)

- [Regulated Pool Supply Plants by Month](https://mpr.datamart.ams.usda.gov/services/v1.1/reports/3359/Supply%20Plants%20by%20Month)

**Raw data:** Each table identifies regulated plants by name, city, state, and the FMMO through which each plant pooled milk during the reported months. Plant function is determined by the table in which the record appears. Supply plants generally perform manufacturing-oriented functions, while distributing plants process and package fluid milk.

`code/Dairy_data_cleaning.ipynb` downloads both tables directly from the USDA AMS Datamart API on its first run. The data were retrieved on July 17, 2026. The notebook keeps December 2025 records for plants located in Michigan, Minnesota, Wisconsin, New York, Pennsylvania, and Vermont and saves the filtered source records in raw/ as:

- `raw/supply_plants_six_states_dec2025.csv`
- `raw/distributing_plants_six_states_dec2025.csv`

The cleaning notebook standardizes the source column names, retains the plant name, city, state, December FMMO order, Class I differential, report information, and publication date, and assigns each plant to one of our regions (Upper Midwest and Northeast). Plant cities are matched to counties using `raw/city_county_crosswalk.csv`, and county FIPS codes are connected to representative county coordinates from `raw/county_centers.csv`. The resulting geocoded plant files are saved in `processed/` as:

- `processed/supply_plants_dec2025.csv`
- `processed/distributing_plants_dec2025.csv`

This procedure identifies 48 supply plants: 40 in the Upper Midwest and 8 in the Northeast. It also identifies 50 distributing plants: 16 in the Upper Midwest and 34 in the Northeast.

The plant symbols indicate the county in which each plant is located rather than its exact street address. When multiple plants are located in the same county, `code/Dairy_DataVisualization2.ipynb` moves their symbols slightly apart so that each plant remains visible.

**Importance for the visualization**: The plant lists allow us to compare the functions of regulated plants located in the two regions. The Upper Midwest has more supply plants, while the Northeast has more distributing plants. This provides context on whether each regulated plant network is oriented more toward manufactured dairy products or fluid-milk bottling. Plant counts do not measure plant size or processing capacity, and the data do not show which plant received milk from a particular county.

### Data Source 3: FMMO Milk Use Shares

The `raw/2025_fmmo_statistics.pdf` file was downloaded from the Central FMMO, and it is available for download here:

[2025 Federal Milk Order Statistics](https://www.fmmacentral.com/PDFdata/sum2025.pdf)

The relevant table was converted to a text file, `raw/2025_fmmo_statistics_extract.txt`, which records the December and annual utilization shares along with the source URL and the cross-checks against the orders' own December publications.

**Raw data**: The report provides 2025 monthly statistics for each Federal Milk Marketing Order, including producer receipts, milk prices, and the percentage of pooled milk utilized in Classes I through IV. Although the report is published on the Central FMMO website, it contains statistics for all federal milk marketing orders. The four utilization classes were presented in the visualization using simplified product labels:

- Class I: fluid milk

- Class II: soft products, including yogurt, sour cream, and ice cream

- Class III: cheese and whey

- Class IV: butter and milk powder

`code/Dairy_data_cleaning.ipynb` parses these from `raw/2025_fmmo_statistics_extract.txt` into `processed/class_utilization_2025.csv`.`code/Dairy_DataVisualization2.ipynb` reads this processed file but uses only the annual weighted-average shares for FMMO 30 (Upper Midwest) and FMMO 1 (Northeast) for the visualization. Annual shares were used because they provide a more representative full-year measure and are less affected by month-to-month changes in pooling and utilization.

**Importance for the visualization**: The utilization shares provide FMMO order-level context for how pooled milk is classified by use. They show that FMMO 30 is strongly concentrated in Class III (cheese and whey), while FMMO 1 distributes pooled milk more evenly across the four classes. These percentages apply to each entire FMMO, not only to milk originating in the six mapped states. They do not show where the milk was physically processed or where the finished products were sold.

### Data Source 4: FMMO Marketing-Area Boundaries

The boundaries of each Federal Milk Marketing Order are defined in federal regulations as lists of counties. We used the definitions for the two orders shown in the visualization, available from the Electronic Code of Federal Regulations:

- [FMMO 30 — Upper Midwest marketing area](https://www.ecfr.gov/current/title-7/part-1030/section-1030.2)
- [FMMO 1 — Northeast marketing area](https://www.ecfr.gov/current/title-7/part-1001)

**Raw data**: Each regulation lists the counties (and in a few cases, parts of counties) that make up the order's marketing area. We copied the county lists for the six study states into a text file,`raw/fmmo_marketing_areas_cfr.txt`, matched each county name to its Census county boundary, and saved the result in `processed/fmmo_marketing_areas.csv`. The visualization draws the outer edge of these counties as the marketing-area boundary.

**Importance for the visualization**: The boundary shows which territory each order regulates. This helps the reader see that farms located outside an order's marketing area can still pool their milk on that order. For example, most of Pennsylvania lies outside the FMMO 1 marketing area, but milk from those counties still pools on FMMO 1. It also shows that the utilization bars describe each order's territory, not the three states as a group.

### Data Source 5: County Boundaries

`raw/us_counties.json`
U.S. Census Bureau county cartographic boundary file in GeoJSON format. `code/Dairy_DataVisualization2.ipynb` uses this file to draw the county shapes in both map panels, to match the marketing-area county lists to their boundaries, and to check that plant symbols are placed on land. Counties are projected to EPSG:5070 (Albers equal-area) before mapping.

### Outputs

- `code/Dairy_data_cleaning.ipynb`\
Reads and organizes the December 2025 FMMO producer-milk records. It combines county records across orders, retains suppressed amounts as state-level totals, and checks the compiled values against the totals published in the original reports.

- `code/Dairy_DataVisualization2.ipynb`\
Combines the processed county milk data with the validated plant locations, FMMO marketing-area boundaries, annual class-utilization shares, and Census county boundaries to produce Visualization 2.

- `processed/compiled_long_dec2025.csv`\
Contains separate county and restricted-total records for each FMMO.

- `processed/county_total_dec2025.csv`\
Contains one record per mapped county after reported milk pounds are summed across the relevant FMMOs.

- `processed/reconciliation_dec2025.csv`\
Compares the extracted and compiled milk values with the order-state totals published in the original reports.

Note: `fo1_dec2025_parsed.csv`, `fo30_dec2025_parsed.csv`, `fo32_dec2025_parsed.csv`, `fo33_mi_dec2025_parsed.csv`, `fo33_nypa_dec2025_parsed.csv`, `fo5_dec2025_parsed.csv` are also written to `processed/` by the cleaning notebook and combined by the compile step.

- `processed/class_utilization_2025.csv`\
  Contains December and annual Class I–IV utilization shares for the relevant FMMOs.

- `outputs/Dairy_DataVisualization2.png`\
  Final two-panel visualization comparing pooled milk, regulated plant functions, and FMMO product utilization in the Upper Midwest and Northeast.

Open Jupyter Notebook and run the two Visualization 2 notebooks in the following order:

- Open `code/Dairy_data_cleaning.ipynb` and select Run All. This creates the processed county milk and reconciliation files.

- Open `code/Dairy_DataVisualization2.ipynb` and select Run All. This creates `outputs/Dairy_DataVisualization2.png`.

The notebooks use paths relative to the repository location, so users should not need to edit local file paths after downloading the repository.
